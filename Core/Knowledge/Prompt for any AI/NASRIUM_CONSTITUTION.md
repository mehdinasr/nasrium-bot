# NASRIUM CONSTITUTION

Version: 1.0

## PURPOSE
NASRIUM shall be developed as a long-term modular software platform.
The project must never depend on any single AI model, chat history, developer or account.

## CORE PRINCIPLES

1. Architecture First
Every implementation follows the architecture before coding.

2. Modular Design
Every module must be independent.

3. No Duplicate Logic
Repeated code or duplicated files are forbidden.

4. Single Responsibility
Each file has one responsibility only.

5. Small Commands
PowerShell commands must remain short, readable and modular.

6. State Driven
Current project state is stored in PROJECT_STATE.json.

7. AI Independent
Any AI must continue development only after loading the Core Knowledge package.

8. Command Tracking
Every important operation receives a unique CMD identifier.

9. Documentation First
Important architectural decisions must be documented.

10. Safe Refactoring
Refactoring must not change project behavior.

11. Backward Compatibility
Existing stable modules must continue working.

12. Recoverability
Project must always be recoverable from Core/Knowledge.

13. Portable Architecture
Entire project knowledge must be portable to another computer or AI.

14. Performance Before Complexity
Prefer simple, maintainable solutions.

15. Constitution Priority
This document has higher priority than implementation details.

END OF CONSTITUTION
