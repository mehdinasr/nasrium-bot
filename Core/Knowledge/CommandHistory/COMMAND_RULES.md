# NASRIUM COMMAND HISTORY RULES

Version: 1.0

Rules:
1. Every important project operation must have a unique CMD ID.
2. CMD files are immutable records.
3. Never overwrite previous CMD files.
4. INDEX.json is the command registry.
5. CURRENT_STATE.json stores latest project position.
6. System must not depend on chat history.
7. Any AI agent can continue project using these files.

Architecture:
NASRIUM SDPA
